Final Project Proposal

Caroline Murray

CRP 558

“Creating a Publicly Available Online Platform for the Saylorville Trail System”

I currently work as a park ranger for the US Army Corps of Engineers at Saylorville Lake in Johnston, IA. Over the past 3 years that I have worked there, numerous members of the public have stopped to ask me if we have hiking or biking trails on our project. We have several recreational trails, however, the methods of information delivery that we use are completely outdated. Currently, we have no hiking trail maps published online with the exception of the paved Neal Smith Bike Trail. In addition, if we have to close sections of the Neal Smith Trail due to high water, the only notice that the public receives is through physical barricades placed on the trail. The public needs better information delivery.

I plan to create a web page with an interactive map of all of the Saylorville recreational trails with mile markers, trail information, trail closures, and accessibility information. The end goal would be to have a page that is easy to perform frequent updates, and would be linked to on the Saylorville Lake web page so that the public have access to up-to-date trail information. The public should be able to open the web page, toggle to choose between hiking and biking trails, zoom in on available facilities near the trails, and see hiking difficulty, trail surface type, and even points of interest with pictures. I plan to include length attributes on every trail layer, and will include a topography basemap as an option to show elevation changes on the trails.

For this project, the data I will use will be collected by myself using ArcCollector to create shapefiles, and have attached pictures. I prefer to use the ArcCollector application because I often use ArcGIS Pro and ArcGIS Online, and they are approved software through the Department of Defense. I plan to embed an ArcGIS Online map into my web page as my interactive map with the different trail layers as toggles. I will also use GitHub to submit the code and create the index.html file. All editing will take place in a brackets file.

I expect to run into issues creating an html file that is easy to edit for other rangers once I move on to another job. I plan to do more research on techniques that coders use to make updating websites simple for those with little or no coding experience. I may need assistance with this! In addition, I need to follow some guidelines for web page design that coordinate with the US Army Corps of Engineers standards – unfortunately in the federal government there are extremely specific requirements for any publication. I expect to run into some issues with my styling especially with these requirements.
